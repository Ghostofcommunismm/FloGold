打印小票 UX 完整规格说明

把"记账完成"这个瞬间，变成一次有仪式感的、可分享的小票打印体验。本文档覆盖状态定义、动画时间线、视觉规范、异常处理与实施建议。

1核心概念 · Concept

为什么做这件事，以及它在产品情绪层面的价值
仪式感
传统记账 App 在「完成」按钮按下后，表单消失，交易出现在列表。整个过程冷冰冰、缺乏反馈。小票打印则把"记录一笔"实体化，变成"我刚刚记下了一件事"的可触摸证据。
差异点
在国内外记账赛道（钱迹、随手记、Money Lover、YNAB、Spendee）中，目前没有一家把这步做成真正"打印"的体验。这是 FloGold 的标志性 feature。
情绪锚点
从「钱出去了」的负罪感，转化为「被记录下来了」的安全感。热敏纸的小票本身就带有"成交/收据"的语义，是金融交互中最经典的情绪载体。
社交货币
小票形态天然适合截图分享。用户可以把"我今天交通花了 128.50"分享到朋友圈/小红书，比单纯一行表格更有传播力。

2状态流程 · State Flow

从打开弹窗到小票收起的四个核心状态
📝
PHASE 1
填写态 IDLE
用户操作中
正常表单：金额、分类、商户、备注。底部「完成」按钮可点击。
🖨️
PHASE 2
打印态 PRINTING
~3.4s
表单 fade-out，机器外壳淡入，纸张从顶部纸槽向下滚出，依次吐出内容。
✓
PHASE 3
完成态 COMPLETE
停留或 tap
小票完整显示，LED 灯转绿，提示"点击任意处关闭"。
📄
PHASE 4
收起态 EXIT
0.5s
小票向上卷起消失，交易卡片在列表中飞入定位。

6边缘场景 · Edge Cases

异常路径的兜底设计，避免用户卡死或体验断裂
正常
✅ 标准流程
用户填好表单 → 完成 → 打印小票 → tap 关闭 → 列表新增
错误
❌ 必填项缺失
「完成」保持 disabled，红边提示 + 抖动反馈。整个流程不会触发打印态。
异常
⚠️ 保存失败（网络/本地）
Phase 2 → Phase 3 直接显示一个 ❌ 错误小票（顶部红色印章），提示「打印失败，请重试」。
降级
🌑 减弱动效
检测到 prefers-reduced-motion，跳过全部印刷动画，直接显示完整小票。
扩展
🖼️ 长按截图分享
Phase 3 长按小票触发 0.3s 反馈动画后，弹出分享菜单（导出 PNG / 微信 / 小红书）。
扩展
🔁 撤销操作
Phase 3 tap 后 5s 内显示 Snackbar「已记录 · 长按撤销」，避免误触。

7实施建议 · Implementation

Vue 3 + Motion-V 落地要点
⚡
Vue 3 状态机
用 `ref` + watch 维护 phase 状态；composable useReceipt() 封装 open/close/print 方法
🎬
Motion-V 编排
用 stagger 让 8 个区块按时间线依次出现；spring 配置 e: 'spring' + stiffness: 200 还原弹性
📐
clip-path 印刷
clip-path: inset(0 0 100% 0) 从底部裁掉到 0%，模拟纸张"吐出"效果
🎨
CSS mask 撕纸边
顶部 / 底部锯齿用 radial-gradient mask 平铺，无需图片资源
📱
触觉反馈
点击完成时调用 navigator.vibrate(8)，打印过程中每 200ms 短振模拟电机
🔊
可选音效
首次完成时播放 0.5s 打印音效（可静音），设置项可关闭
