设计理念

原版痛点：LCD 屏幕塞下了 6 段内容（收支切换 / 金额 / 分类 / 子分类 / 三行商户信息 / 完成按钮），层级全靠字号和虚线区分，缺少真实 POS 机的"机壳 ↔ 屏幕"层次感。收支分段按钮 13px 的字号在小屏幕上显得过于突出。

主要优化动作

  • 屏幕内只保留"金额 + 当前分类"两行核心信息
  • 收支切换、分类选择、商户备注全部移到机身"实体按钮"区域
  • 屏幕高度从 280px 缩到 130px，让"POS 机壳"更突出
  • LCD 字号从 26px 提到 30px 提升主信息可读性
  • 真实 POS 机的"侧键、磁卡槽、铭牌"成为新信息载体

关键参数

屏幕信息密度高（拥挤）
金额字号26px
收支按钮13px / 屏内
分类入口4 个分页 / 屏内
商户输入3 行 Input / 屏内
